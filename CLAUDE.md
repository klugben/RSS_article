# CLAUDE.md - 项目指南

这是一个基于认知敏捷法(42COG)的AI原生项目初始化模板。

## 开发环境

- **设备**：搭载 Apple 芯片的 Mac 电脑
- **Node.js 管理**：通过 bun 进行安装管理
- **Python 环境**：通过 uv 配置
- **Git 代码托管平台**：按项目需要配置

## 项目结构

```
/
├── .42cog/          # 认知敏捷法配置目录
│   ├── meta/            # 项目元信息
│   ├── real/            # 现实约束
│   ├── cog/             # 认知模型
│   ├── spec/            # 规约文档
│   ├── work/            # 工作记录
│   └── others/          # 其他文档
├── .42plugin/       # 插件技能库
├── .claude/         # Claude Code 配置目录
├── docs/            # 文档目录
├── notes/           # 笔记目录
├── source/          # 源文件（不纳入版本控制）
├── spec/            # 规格说明
└── src/             # 源代码
```

## 开发规范

1. **代码风格**
- 使用有意义的变量和函数命名
- 保持代码简洁、可读
- 遵循项目既有的代码风格

2. **初始化项目元信息**
   编辑 `.42cog/meta/meta.md`，填写项目基本信息

3. **定义现实约束**
   编辑 `.42cog/real/real.md`，列出项目的现实约束（最多7条）

4. **创建认知模型**
   编辑 `.42cog/cog/cog.md`，定义项目的核心实体和关系

5. **生成规约文档**
   使用 `.42plugin/42edu/` 下的技能生成各类规约文档

6. **开始编码**
   在 `src/` 目录下开发代码


## 认知敏捷法 (42COG)

### RCSW 工作流

```
Real (现实约束) → Cog (认知模型) → Spec (规约文档) → Work (实际作品)
```


## 注意事项

- 请勿提交敏感信息（如 API 密钥、密码等）
- `source/` 目录不纳入版本控制
- 在进行重大更改前，请确保与团队成员沟通


## 每次启动前,默认先确认任务执行进度,阅读项目背景信息
  - `Readme.md`
  - 项目规约:`.42cog/`
  - 项目规约:`.spec/`
  - 执行进度报告,地址在 `.42cog/work/`，文件名格式：`yyyy-mm-dd hh-mm [操作总结].md`(按需,每次阅读项目计划,和最新进展即可)
  - 对话历史报告,在`.chats/hi+ai/`(可选,按需,每次阅读最新对话即可)


3. **Git 提交**
   - 执行 `/chat_output_git-commit` 作为收尾
