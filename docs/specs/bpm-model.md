# BPM 业务流程模型

> 项目: RSS_article
> 状态: 已确认

## 主流程：每日简报生成与推送

```
[开始]
   │
   ▼
┌─────────────────────┐
│ T1: RSS 抓取         │
│ lumen fetch          │
│ 从订阅源获取新文章    │
│ 写入 feeds.db        │
└────────┬────────────┘
         │
         ▼
┌─────────────────────┐
│ T2: 定时触发          │
│ OpenClaw Cron        │
│ 每日 8:00 触发 Agent  │
└────────┬────────────┘
         │
         ▼
┌─────────────────────┐
│ T3: 数据查询          │
│ Agent 读取 feeds.db  │
│ 查询最近 24h 新文章   │
│ (只读操作)           │
└────────┬────────────┘
         │
         ▼
     ┌───┴───┐
     │有文章？│
     └───┬───┘
     Yes │  │ No
         │  ▼
         │  [结束 - 不推送]
         ▼
┌─────────────────────┐
│ T4: 主题分类          │
│ 按关键词规则匹配主题   │
│ 7 个预定义主题 + 未分类│
└────────┬────────────┘
         │
         ▼
┌─────────────────────┐
│ T5: 摘要生成          │
│ 提炼核心观点          │
│ ≤3篇: 逐篇摘要        │
│ >3篇: 要点+链接列表   │
│ 总字数 ≤ 400 字       │
└────────┬────────────┘
         │
         ▼
┌─────────────────────┐
│ T6: 消息推送          │
│ openclaw message send │
│ --channel feishu      │
│ 推送到飞书群组         │
└────────┬────────────┘
         │
         ▼
       [结束]
```

## 任务说明

| 任务 | 执行者 | 频率 | 输入 | 输出 |
|------|--------|------|------|------|
| T1 RSS 抓取 | lumen (外部) | 持续/定时 | RSS feed URL | feeds.db 新记录 |
| T2 定时触发 | OpenClaw Cron | 每日 8:00 | Cron 配置 | Agent 调用 |
| T3 数据查询 | Agent (AI) | Cron 触发后 | feeds.db | 文章列表 |
| T4 主题分类 | Agent (AI) | 查询后 | 文章标题/正文 | 主题标签 |
| T5 摘要生成 | Agent (AI) | 分类后 | 分类后文章 | 简报文本 (≤400字) |
| T6 消息推送 | OpenClaw | 生成后 | 简报文本 | 飞书群消息 |

## 异常流程

### E1: 数据库查询失败

```
T3 执行 SQL 查询失败
        │
        ▼
  检查 DB_PATH 是否存在
        │
    ┌───┴───┐
    │存在？  │
    └───┬───┘
   Yes │  │ No
       │  ▼
       │  输出错误日志
       │  [结束 - 推送失败]
       ▼
  重试查询（只读，安全）
       │
    ┌───┴───┐
    │成功？  │
    └───┬───┘
   Yes │  │ No
       │  ▼
       │  [结束 - 推送失败]
       ▼
  继续 T4
```

### E2: Gateway 异常（Cron 未触发）

```
用户发现 8:00 后未收到简报
        │
        ▼
  SSH 到服务器检查 Gateway 状态
        │
    ┌──────────┴──────────┐
    │Gateway 正常？        │
    └──────────┬──────────┘
          Yes  │      │ No
               │      ▼
               │  重启 Gateway
               │      │
               │      ▼
               │  手动触发推送
               ▼
         检查 Cron 配置
         检查 Agent 状态
```

### E3: SQLite 并发冲突

```
T3 执行查询时 feeds.db 被锁定
        │
        ▼
  SQLite WAL 模式自动处理读写并发
  （只读查询不阻塞写入）
        │
    ┌───┴───┐
    │成功？  │
    └───┬───┘
   Yes │  │ No (BUSY)
       │  ▼
       │  等待 1 秒后重试
       │  最多重试 3 次
       ▼
  继续 T4
```

## 补充流程：多订阅源独立推送

```
Cron 触发 / 手动触发
        │
        ▼
  查询所有活跃订阅源
        │
        ▼
  FOR EACH feed:
        │
        ▼
  查询该 feed 最近 24h 新文章
        │
        ▼
  过滤已推送文章（去重）
        │
    ┌───┴───┐
    │有新文？│
    └───┬───┘
    Yes │  │ No
        │  ▼
        │  跳过该 feed
        ▼
  生成该 feed 的独立简报（≤400字）
        │
        ▼
  推送到飞书群组
        │
        ▼
  记录已推送文章 ID
        │
        ▼
  NEXT feed
```

## 补充流程：手动触发推送

```
用户执行 openclaw agents run RSS_article
        │
        ▼
  Agent 执行与 Cron 相同的逻辑
  （查询 → 去重 → 分类 → 摘要 → 推送）
        │
        ▼
  推送完成
```

## 业务规则

| # | 规则 | 说明 |
|---|------|------|
| BR-1 | 只读查询 | Agent 对 feeds.db 只执行 SELECT（去重记录可写入独立表或文件） |
| BR-2 | 24 小时窗口 | 只查询 published_at >= now() - 1 day 的文章 |
| BR-3 | 400 字硬限制 | 简报总字数（含标题和标签）不得超过 400 字 |
| BR-4 | 无文章不推送 | 当日无新文章时跳过推送，不发送"无新文章"消息 |
| BR-5 | 忠于原文 | 摘要不编造信息、不做主观评价 |
| BR-6 | 时区转换 | published_at 从 UTC 转换为北京时间 (UTC+8) |
| BR-7 | 已推送去重 | 记录已推送文章 ID，同一篇文章不重复推送 |
| BR-8 | 每源独立日报 | 每个订阅源独立生成一份简报，不合并多源 |
| BR-9 | 支持手动触发 | 除 Cron 外，支持 openclaw agents run 手动触发 |
