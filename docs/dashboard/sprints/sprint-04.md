## 问题背景

用户反馈：手动触发和定时触发的日报里，有时会丢失文章主要内容总结。

## 根因分析

| # | 根因 | 说明 |
|---|------|------|
| 1 | >3 篇切换为"链接列表"模式 | SOUL.md 规定 >3 篇时只输出"要点概括+链接列表"，单篇摘要完全丢弃 |
| 2 | 400 字总限制过紧 | 老布平均文章 7879 字，3 篇压缩到 400 字极度精简 |
| 3 | 无单篇摘要最低保障 | 规则未要求每篇文章必须有内容总结 |

## 变更详情

### SOUL.md 变更

1. **字数限制**：`≤400 字` → `≤800 字`
2. **统一格式**：取消两套格式（≤3 篇 / >3 篇），统一为每篇文章必有 1-2 句摘要 + 原文链接
3. **摘要质量规则**：
   - 每篇摘要必须包含核心论点、关键数据、或可行动洞察
   - 禁止仅重复标题或使用"介绍了...""讲述了..."等无信息量表述
   - 文章过多时按主题重要性取舍，末尾标注"另有 N 篇文章详见原文链接"
4. **约束新增**：每篇文章必须有实质性摘要，禁止仅输出链接

### USER.md 变更

1. 飞书推送长度：`≤400 字` → `≤800 字`
2. 字数与格式描述同步更新

## 验收标准

- [ ] SOUL.md 规则清晰、无矛盾
- [ ] USER.md 与 SOUL.md 一致
- [ ] 手动触发后，每篇文章都有实质性摘要（非空、非标题重复）
- [ ] 总字数 ≤800 字
- [ ] 多篇文章时按主题分组、每篇有链接
- [ ] 无新文章时仍正确跳过推送
